Enfoques para el módulo circular Arch

3 opciones de implementación CSS — mismo módulo custom/arch existente

A

CSS puro con dimensiones fijas

Definir width y height iguales + border-radius: 50%. Simple, predecible, funciona siempre.

Pros

  • Máxima simplicidad
  • Círculo perfecto garantizado
  • Fácil de ajustar tamaño

Cons

  • Tamaño fijo (no se adapta si cambia font-size global)
B

CSS con aspect-ratio

Usar aspect-ratio: 1 + border-radius: 50% para que se mantenga circular sin fijar ambas dimensiones.

Pros

  • Se adapta al contenido
  • Moderno y limpio

Cons

  • GTK3 (usado por Waybar) tiene soporte limitado de aspect-ratio
  • Puede no funcionar correctamente
C

CSS con padding simétrico

Usar padding igual en todos los lados + border-radius: 50%. El círculo se forma alrededor del icono naturalmente.

Pros

  • Se adapta al tamaño del icono
  • Compatibilidad GTK3 total
  • Consistente con el patrón existente de padding en los módulos

Cons

  • Si el contenido no es cuadrado, el círculo puede deformarse ligeramente

Recomendación: Opción A

Dimensiones fijas es lo más confiable con GTK3/Waybar. El icono Arch tiene tamaño fijo (font-size), así que no hay razón para hacerlo responsive. Además es lo más simple de mantener y extender después.
